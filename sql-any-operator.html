01 — Pengertian

Apa itu ANY?

ANY adalah operator SQL yang digunakan bersama operator perbandingan (=, <>, <, >, <=, >=) untuk membandingkan satu nilai dengan kumpulan nilai yang dihasilkan sub-query. Kondisi bernilai TRUE jika minimal satu nilai dalam kumpulan memenuhi syarat.

SQL Syntax
SELECT column_name(s)
FROM table_name
WHERE nilai_angka > ANY (
    SELECT nilai_angka
    FROM nilai
    WHERE kode_matakuliah = 'ASD'
); -- TRUE jika > setidaknya SATU nilai dalam set
"

Bayangkan seleksi lomba. Panitia bilang: "Kamu lolos kalau nilaimu lebih tinggi dari setidaknya satu peserta di kelas ASD."

Kamu tidak harus mengalahkan semua orang. Kalahkan si paling lemah saja → kamu lolos.
Itulah > ANY — standarnya rendah, yang penting ada satu yang bisa kamu lewati.

Fungsi ANY sebagai filter

ANY ada di dalam klausa WHERE. Setelah evaluasi, hasilnya sederhana:

TRUE
Baris masuk ke hasil query. Kondisi terpenuhi untuk setidaknya satu nilai dalam sub-query.
FALSE
Baris dibuang. Tidak ada satupun nilai dalam sub-query yang memenuhi kondisi.
02 — Mekanisme

Cara Kerja

◈
Setup: Sub-query kelas ASD menghasilkan set nilai {60, 70, 80}. Outer query akan mengecek setiap mahasiswa satu per satu.
1
Sub-query dieksekusi lebih dulu
Database menjalankan sub-query dan menyimpan hasilnya sebagai set sementara: {60, 70, 80}. Sub-query ini hanya dieksekusi sekali.
2
Outer query baca baris pertama
Misal: Aldi dengan nilai_angka = 55. Database akan mengevaluasi kondisi WHERE untuk baris ini.
3
Evaluasi satu per satu terhadap set
Database cek: 55 > 60? Tidak. 55 > 70? Tidak. 55 > 80? Tidak.
Semua FALSE → hasil ANY = FALSE → baris Aldi dibuang.
4
Lanjut ke baris berikutnya
Budi dengan nilai_angka = 65: 65 > 60? Ya! → langsung TRUE.
ANY berhenti cek begitu ada satu yang TRUE → baris Budi masuk.
5
Ulangi untuk semua baris
Proses berlanjut sampai semua baris di outer query dievaluasi. Hasil akhir = kumpulan baris yang lolos filter WHERE.

04 — Praktik

Studi Kasus

Database: mahasiswa(nim, nama) | nilai(nim, kode_matakuliah, nilai_huruf, nilai_angka)
Kasus 01
Mahasiswa dengan nilai di atas minimal kelas ASD

Tampilkan mahasiswa yang nilai angkanya lebih besar dari setidaknya satu nilai di kelas ASD. Artinya: nilai kamu harus mengalahkan si paling lemah di ASD.

SQL
SELECT nim, nama, nilai_angka
FROM nilai
WHERE nilai_angka > ANY (
    SELECT nilai_angka
    FROM nilai
    WHERE kode_matakuliah = 'ASD'
);
-- Ekuivalen: WHERE nilai_angka > MIN(nilai ASD)
-- Jika ASD = {60,70,80} → lolos jika nilai > 60
→
Hasil: Budi (65), Citra (75), Dani (80), Eva (90) — semua yang di atas 60.
Kasus 02
Mahasiswa yang nilainya ada di daftar nilai kelas ASD

Tampilkan mahasiswa yang nilai angkanya sama persis dengan salah satu nilai di kelas ASD. Operator = ANY identik dengan IN.

SQL
SELECT nim, nama, nilai_angka
FROM nilai
WHERE nilai_angka = ANY (
    SELECT nilai_angka
    FROM nilai
    WHERE kode_matakuliah = 'ASD'
);
-- Sama persis dengan:
-- WHERE nilai_angka IN (SELECT nilai_angka FROM nilai WHERE kode_matakuliah = 'ASD')
→
Hasil: Hanya Dani (80) — satu-satunya yang nilainya ada dalam set {60, 70, 80}.
Kasus 03
Mahasiswa yang nilainya di bawah nilai tertinggi kelas ASD

Tampilkan mahasiswa yang nilainya lebih kecil dari setidaknya satu nilai di ASD. Artinya: nilaimu harus di bawah si paling tinggi di ASD.

SQL
SELECT nim, nama, nilai_angka
FROM nilai
WHERE nilai_angka < ANY (
    SELECT nilai_angka
    FROM nilai
    WHERE kode_matakuliah = 'ASD'
);
-- Ekuivalen: WHERE nilai_angka < MAX(nilai ASD)
-- Jika ASD = {60,70,80} → lolos jika nilai < 80
→
Hasil: Aldi (55), Budi (65), Citra (75) — semua yang di bawah 80. Dani dan Eva tidak masuk karena nilainya ≥ 80.
◎
Ingat kuncinya: ANY itu filter. TRUE → baris masuk ke hasil. FALSE → baris dibuang. ANY berhenti evaluasi begitu menemukan satu nilai yang cocok — efisien dan cepat.

ANY = longgar (cukup satu). ALL = ketat (harus semua). Itu saja bedanya.
